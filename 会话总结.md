# 博客项目会话总结

本文档用于新会话快速恢复上下文。当前项目是一个 **Nuxt 4 全栈动态博客**，核心目标是：

```text
后台发文章 -> 写入 PostgreSQL -> 前台 SSR/API 实时读取 -> 不依赖重新构建
```

当前实际工作目录：

```bash
/home/jiupan/blog/blog-nuxt
```


重要版本约束：

- Prisma 固定在 `6.19.3`，不要直接升级到 Prisma 7。
- `nuxt-security` 固定在 `2.5.0`，避免 Node 版本要求问题。
- 容器内执行 Prisma 命令时显式指定版本：`npx --package prisma@6 prisma ...`。

## 重要注意事项

### 长期维护规则：数据库表结构变更必须写 migration

涉及数据库表、字段、索引、枚举、关系的任何变更，必须同时维护：

```text
prisma/schema.prisma
prisma/migrations/<timestamp>_<name>/migration.sql
```

不要只改 `schema.prisma`。生产环境执行的是：

```bash
npx --package prisma@6 prisma migrate deploy
```

它只会执行 `prisma/migrations/**/migration.sql`，不会根据 `schema.prisma` 自动补表。

本项目曾出现过 `schema.prisma` 里已有 `Setting` model，但 migration 没创建 `"Setting"` 表，导致生产环境报错：

```text
The table `public.Setting` does not exist in the current database.
```

修复方式是新增 migration：

```text
prisma/migrations/20260701150000_add_missing_setting_table/migration.sql
```

以后新增功能时，如果需要新增或修改模型，必须先补 migration，再提交部署。典型流程：

```bash
npx prisma migrate dev --name 描述性名称
git add prisma/schema.prisma prisma/migrations
git commit -m "add xxx migration"
```

如果本地库因为历史占位 init migration 出现 drift，不能随便 reset 生产数据；先确认当前库和目标 SQL，再采用明确 migration 或 `prisma db execute` 处理。

### 长期维护规则：生产库禁止用 db push 代替 migration

生产环境不要用：

```bash
prisma db push
```

`db push` 会绕过 migration 历史，不适合生产排查和回溯。生产统一使用：

```bash
docker compose -f docker-compose.server.yml exec -T app npx --package prisma@6 prisma migrate deploy
```


## 服务器更新方案

### 1. 本地打包镜像

## Nginx 和 HTTPS

域名：

```text
blog.ittx.cn
```

Nuxt app 默认监听：

```text
127.0.0.1:3000 或 0.0.0.0:3000
```

Nginx 反代到 app：

```nginx
server {
    listen 80;
    server_name blog.ittx.cn;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

HTTPS：

```bash
sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d blog.ittx.cn
```

阿里云安全组需要放行：

```text
TCP 80
TCP 443
```
