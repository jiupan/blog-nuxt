核心目标：

```text
后台发文章 → 写入数据库 → 前台 SSR 实时读取 → 不需要重新构建
```

Nuxt 官方现在把 Nuxt 定位为 **Full-Stack Vue Framework**，底层由 **Vue + Nitro** 支撑，适合做 SSR、API Routes、全栈应用和 SEO 友好的网站。Nuxt 4 的 `server/` 目录可以自动注册 API、server routes 和 middleware，`server/api` 下的文件会变成 `/api/*` 接口。([Nuxt](https://nuxt.com/?utm_source=chatgpt.com))

# 二、整体架构

```text
浏览器
  ↓
Nginx
  ↓
Nuxt 4 SSR App
  ├── app/pages       前台页面 + 后台页面
  ├── server/api      后端接口
  ├── server/utils    数据库、认证、Markdown 工具
  ↓
Prisma
  ↓
PostgreSQL
```

发布文章流程：

```text
/admin/posts/create
  ↓
POST /api/admin/posts
  ↓
Prisma 写入 PostgreSQL
  ↓
用户访问 /posts/[slug]
  ↓
Nuxt SSR 查询数据库
  ↓
返回最新 HTML
```

所以你要的效果可以实现：

```text
发文章不用重新 build
发文章不用重新 generate
刷新页面即可看到新文章
```

Nuxt 4 支持通过 Nitro 部署到 Node.js 服务，也支持 `routeRules` 做不同路由的渲染和缓存策略。([Nuxt](https://nuxt.com/docs/4.x/getting-started/server?utm_source=chatgpt.com))

# 四、数据库设计

第一版表不要太多，重点保证博客核心闭环。

```prisma
model User {
  id           Int      @id @default(autoincrement())
  username     String   @unique
  passwordHash String
  role         Role     @default(ADMIN)
  createdAt    DateTime @default(now())
  updatedAt    DateTime @updatedAt
}

model Post {
  id          Int        @id @default(autoincrement())
  title       String
  slug        String     @unique
  summary     String?
  content     String
  cover       String?
  status      PostStatus @default(DRAFT)
  viewCount   Int        @default(0)

  categoryId  Int?
  category    Category?  @relation(fields: [categoryId], references: [id])

  tags        PostTag[]

  seoTitle       String?
  seoDescription String?

  createdAt   DateTime   @default(now())
  updatedAt   DateTime   @updatedAt
  publishedAt DateTime?

  @@index([slug])
  @@index([status])
  @@index([publishedAt])
  @@index([categoryId])
}

model Category {
  id        Int      @id @default(autoincrement())
  name      String
  slug      String   @unique
  posts     Post[]
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

model Tag {
  id        Int       @id @default(autoincrement())
  name      String
  slug      String    @unique
  posts     PostTag[]
  createdAt DateTime  @default(now())
  updatedAt DateTime  @updatedAt
}

model PostTag {
  postId Int
  tagId  Int

  post Post @relation(fields: [postId], references: [id], onDelete: Cascade)
  tag  Tag  @relation(fields: [tagId], references: [id], onDelete: Cascade)

  @@id([postId, tagId])
}

enum PostStatus {
  DRAFT
  PUBLISHED
  ARCHIVED
}

enum Role {
  ADMIN
}
```

后期再扩展：

```text
Comment        评论
FriendLink     友链
Project        项目展示
VisitLog       访问记录
SiteSetting    站点配置
PostRevision   文章历史版本
```

第一版不要做太多，不然很容易做不完。

# 七、渲染与缓存策略

`nuxt.config.ts` 建议这样：

```ts
export default defineNuxtConfig({
  modules: [
    '@nuxt/ui',
    '@nuxt/image',
    '@nuxtjs/sitemap',
    'nuxt-auth-utils',
    'nuxt-security'
  ],

  routeRules: {
    '/': { swr: 300 },
    '/posts': { swr: 300 },
    '/posts/**': { swr: 300 },
    '/categories/**': { swr: 300 },
    '/tags/**': { swr: 300 },
    '/archive': { swr: 300 },

    '/admin/**': { ssr: false },
    '/api/admin/**': { cors: false }
  }
})
```

解释一下：

```text
前台页面：SSR + SWR 缓存
后台页面：CSR
后台接口：必须登录
```

这样既可以 SEO，又能保证文章发布后不用重新构建。

# 八、关键功能实现方案

## 1. 认证登录

最终方案：

```text
nuxt-auth-utils + HttpOnly Cookie Session
```

流程：

```text
/admin/login 输入账号密码
  ↓
POST /api/auth/login
  ↓
Prisma 查询 User
  ↓
Argon2id 校验密码
  ↓
setUserSession 写入加密 Cookie
  ↓
跳转 /admin
```

`nuxt-auth-utils` 使用 secured & sealed cookies 存 session，不需要额外建 session 表，适合你这种单人博客后台。([Nuxt](https://nuxt.com/docs/4.x/guide/recipes/sessions-and-authentication?utm_source=chatgpt.com))

后台路由保护：

```text
app/middleware/admin-auth.ts
server/middleware/auth.ts
```

前端页面防止未登录访问 `/admin/**`，后端接口防止未登录调用 `/api/admin/**`。

```text
一个基于 Nuxt 4 的全栈动态个人博客系统，支持 SSR、后台发文、Markdown 编辑、分类标签、图片上传、SEO、sitemap 和 Docker 部署。文章存储在 PostgreSQL 中，发布后前台实时读取，无需重新构建。
```

这套比 **Halo + Astro SSG** 更符合你现在的需求，因为它解决了你最烦的点：**发文章不用重新构建**。
