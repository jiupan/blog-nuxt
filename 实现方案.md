# Nuxt 动态博客实现方案

本文档描述当前项目的实际实现方案，不再是初始化阶段的纯规划。核心目标：

```text
后台发文章 -> 写入数据库 -> 前台 SSR/API 实时读取 -> 不需要重新构建
```

项目定位：

```text
一个基于 Nuxt 4 的全栈动态个人博客系统，支持 SSR、后台发文、Markdown 编辑、分类标签、菜单、图片上传、图库、SEO、sitemap、AI 辅助创作、RAG 站内问答和 Docker Compose 部署。
```

## 一、当前技术栈

| 模块 | 当前选择 | 说明 |
| --- | --- | --- |
| 框架 | Nuxt 4.4.8 | 前台页面、后台页面、SSR、后端 API |
| 前端 | Vue 3 + TypeScript | Nuxt 4 默认技术基础 |
| 服务端 | Nitro + server/api | 登录、文章、上传、搜索、AI、RAG 等接口 |
| UI | Nuxt UI 4.9.0 + Tailwind CSS | 后台管理和部分前台交互 |
| 图片 | Nuxt Image 2.0.0 + sharp | 图片组件和服务端图片能力 |
| 数据库 | PostgreSQL + pgvector | 文章数据、后台配置、向量索引 |
| ORM | Prisma 6.19.3 | 类型化数据库访问 |
| 认证 | nuxt-auth-utils | HttpOnly Cookie session |
| 密码 | Argon2 | 管理员密码哈希校验 |
| Markdown 编辑 | md-editor-v3 | 后台 Markdown 编辑器 |
| Markdown 渲染 | markdown-it + Shiki + sanitize-html | 前台渲染、代码高亮、防 XSS |
| SEO | @nuxtjs/sitemap + useSeoMeta + robots | sitemap 和页面元信息 |
| 安全 | nuxt-security 2.5.0 | 安全 Header，当前关闭内置 rateLimiter |
| AI | OpenAI-compatible Chat/Embedding/Rerank API | 默认 DeepSeek 文本模型，Embedding/Rerank 可配置 |
| 部署 | Docker Compose | 生产 app + pgvector postgres |

版本约束：

- Prisma 保持 `6.19.3`，生产命令使用 `npx --package prisma@6 prisma ...`。
- `nuxt-security` 保持 `2.5.0`。
- RAG 依赖 pgvector，生产数据库镜像应为 `pgvector/pgvector:pg16` 或等价环境。

## 二、整体架构

```text
浏览器
  ↓
Nuxt 4 SSR/CSR App
  ├── app/pages        前台页面 + 后台页面
  ├── app/components   业务组件
  ├── server/api       后端接口
  ├── server/services  AI、RAG、SEO 检查、链接检查等服务
  └── server/utils     数据库、认证、Markdown、菜单、响应工具
  ↓
Prisma
  ↓
PostgreSQL + pgvector
```

发文流程：

```text
/admin/posts/create
  ↓
POST /api/admin/posts
  ↓
Prisma 写入 PostgreSQL
  ↓
用户访问 /posts/[slug] 或 /[slug]
  ↓
Nuxt SSR/API 查询数据库
  ↓
返回最新内容
```

AI 问答流程：

```text
已发布文章
  ↓
POST /api/admin/ai/index/rebuild
  ↓
分块、生成 embedding、写入 PostChunk
  ↓
POST /api/ai/ask
  ↓
向量检索 + 可选 rerank + 大模型回答 + 来源引用
```

## 四、数据库设计现状

当前模型：

```text
User
Post
Category
Tag
PostTag
PostRelation
PostChunk
Setting
Menu
MenuItem
```

关键关系：

- `Post` 可关联一个 `Category` 和多个 `Tag`。
- `Category` 支持 `icon`。
- `PostRelation` 表示文章之间的关联关系，支持 AI 和人工来源。
- `PostChunk` 保存文章分块、embedding、索引状态，用于 RAG 检索。
- `Setting` 是 key-value 站点配置。
- `Menu` / `MenuItem` 支持多位置菜单和未来多级菜单。

当前枚举：

```text
PostStatus: DRAFT / PUBLISHED / ARCHIVED
PostChunkStatus: ACTIVE / STALE
PostRelationType: PREREQUISITE / EXTENSION / SAME_TOPIC / PRACTICE / BACKGROUND
PostRelationSource: AI / MANUAL
Role: ADMIN
MenuLocation: PRIMARY / FOOTER / MOBILE / CUSTOM
MenuItemType: CUSTOM / POST / CATEGORY / TAG / PAGE / ARCHIVE / HOME
```

当前 migrations：

```text
20260627183138_init
20260629170000_add_menus
20260701150000_add_missing_setting_table
20260701162000_add_category_icon
20260704123000_add_post_relations
20260704143000_add_post_chunks_pgvector
```

维护规则：

```text
任何 schema 变更必须同时维护 migration。
生产只用 prisma migrate deploy。
不要用 prisma db push 修生产库。
```

## 七、渲染、安全和配置

当前 `nuxt.config.ts` 关键配置：

```ts
export default defineNuxtConfig({
  compatibilityDate: '2026-06-28',
  srcDir: 'app',
  modules: [
    '@nuxt/ui',
    '@nuxt/image',
    '@nuxtjs/sitemap',
    'nuxt-auth-utils',
    'nuxt-security'
  ],
  routeRules: {
    '/admin/**': { ssr: false },
    '/api/admin/**': { cors: false }
  },
  sitemap: {
    sources: ['/api/__sitemap__/urls']
  }
})
```

当前没有给前台页面配置 SWR routeRules；前台保持默认 SSR/API 行为。

Session 配置：

```text
name: __Host-nuxt-session
maxAge: 7 天
httpOnly: true
secure: true
sameSite: lax
path: /
```

登录保护：

- 前端：`app/middleware/admin-auth.ts` 保护后台页面。
- 后端：后台接口调用 `requireAdmin(event)`。
- 登录接口：Argon2 校验密码，失败时做内存级 15 分钟 5 次限流。

安全策略：

- Markdown 禁止原始 HTML，并 sanitize 输出。
- 上传接口限制类型和大小。
- `nuxt-security` 启用基础 headers，但当前 `rateLimiter: false`。
- 生产 `.env` 不入库，部署脚本校验必要变量。

## 八、核心业务实现

### 文章发布

后台文章表单字段包括：

```text
title
slug
summary
cover
categoryId
tagIds
content
status
seoTitle
seoDescription
```

发布逻辑：

```text
DRAFT: 草稿，不作为公开文章展示
PUBLISHED: 已发布，公开接口可读取
ARCHIVED: 归档状态
```

公开文章查询通常应满足：

```text
status = PUBLISHED
publishedAt <= now()
```

### Markdown

后台使用 `md-editor-v3` 编辑。前台详情页展示服务端渲染结果：

```text
Markdown 原文
  ↓
markdown-it
  ↓
Shiki 代码高亮
  ↓
sanitize-html
  ↓
TOC / 字数 / 阅读时间 / HTML
```

### 菜单

菜单数据在数据库：

```text
Menu
MenuItem
```

前台主导航读取 `PRIMARY` 菜单，无数据时兜底：

```text
文库 /posts
专栏 /archive
友链 /link
我的 /about
```

### 站点设置

`Setting` 表以 key-value 存储。当前关键设置：

```text
site_title
site_subtitle
sidebar_description
site_logo
site_favicon
seo_noindex
seo_keywords
seo_description
```

`site_favicon` 已用于前台品牌图标回显。

### 图片和图库

上传规则：

```text
允许 jpg / jpeg / png / webp / gif
最大 50MB
按年月目录保存
URL 通过 /uploads/** 访问
```

当前仍是本地 volume 方案，后续可替换为 OSS/COS/R2/MinIO。

## 九、AI 和 RAG 方案

当前 AI 能力分三类。

内容生成/分析：

```text

写作助手
SEO 检查
站点洞察
草稿整理
月度回顾
链接检查
相关文章推荐
```

RAG 检索：

```text
已发布文章 -> chunkPost -> embedTexts -> PostChunk.embedding -> 检索 -> 可选 rerank -> askBlog 生成回答
```

相关服务：

```text
server/utils/ai.ts

server/services/rag/chunker.service.ts
server/services/rag/indexer.service.ts
server/services/rag/retrieval.service.ts
server/services/rag/reranker.service.ts
server/services/rag/ask.service.ts
server/services/rag/vector-utils.ts
```

环境变量：

```env
AI_API_KEY=""
AI_BASE_URL="https://api.deepseek.com"
AI_MODEL="deepseek-v4-flash"
AI_EMBEDDING_API_KEY=""
AI_EMBEDDING_BASE_URL="https://api.openai.com/v1"
AI_EMBEDDING_MODEL="text-embedding-3-small"
AI_EMBEDDING_DIMENSIONS="1536"
AI_RERANK_ENABLED="false"
```

约束：

- `PostChunk.embedding` migration 当前使用 `vector(1536)`，embedding 维度变更需要同步 migration 和索引策略。
- RAG 索引需要手动调用重建接口或后续补自动更新逻辑。
- AI API Key 不允许提交到仓库。

## 十、部署方案

生产 compose 当前是两个服务：

```text
postgres: pgvector/pgvector:pg16
app:      APP_IMAGE
```

`docker-compose.server.yml` 关键点：

```text
postgres-data volume 保存数据库
uploads volume 保存上传文件
app 暴露 3000
UPLOAD_DIR=/app/uploads
```

`deploy.sh` 当前流程：

```text
读取 .env
校验必要环境变量
禁止 Docker 生产 DATABASE_URL 指向 localhost/127.0.0.1
docker compose pull app
docker compose up -d app
pg_isready 检查数据库
npx --package prisma@6 prisma migrate deploy
docker compose ps
```

生产 `.env` 必要变量：

```text
APP_IMAGE
POSTGRES_DB
POSTGRES_USER
POSTGRES_PASSWORD
DATABASE_URL
NUXT_SESSION_PASSWORD
ADMIN_USERNAME
ADMIN_PASSWORD
SITE_URL
SITE_NAME
```

生产敏感配置只放服务器，例如：

```text
/opt/blog-nuxt/.env
```

```bash
npm install
```

```bash

```

```bash
npm run dev
```

Prisma：

```bash
npx prisma validate

```

```bash
docker compose -f docker-compose.server.yml exec -T app npx --package prisma@6 prisma migrate deploy
```

注意：当前本地库历史上可能因为旧 init migration 占位文件出现 drift。不要为了消除 drift 随意 reset 有价值数据。

## 十二、后续演进建议

优先级较高：

```text
1. 给 RAG 索引增加文章发布/更新后的自动增量索引。
2. 明确 /api/ai/** 哪些需要公开，哪些需要管理员权限，并统一路由命名。
3. 给 AI/RAG 接口增加更稳定的错误提示和超时控制。
4. 为 PostRelation 和 PostChunk 增加基础测试或脚本级验证。
5. 为生产部署补充 pgvector 迁移前检查。
```

可延后：

```text
RSS
评论
友链数据库化
访问统计
文章修订历史
对象存储
OG Image 自动生成
结构化数据 JSON-LD
```

最终路线保持不变：

```text
Nuxt 4 + PostgreSQL + Prisma + SSR + 后台管理 + AI/RAG 增强
```

这套方案的核心价值是：文章、菜单、设置和 AI 索引都以数据库为中心，内容更新不依赖重新构建；Nuxt 负责 SSR、后台管理和 API，PostgreSQL/pgvector 负责结构化数据与语义检索。
