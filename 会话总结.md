本文档用于新会话快速恢复上下文。当前项目是一个 **Nuxt 4 全栈动态博客**，核心目标仍然是：

```text
后台发文章 -> 写入 PostgreSQL -> 前台 SSR/API 实时读取 -> 不依赖重新构建
```

## 当前架构治理结果

当前治理计划 `架构治理计划.md` 中阶段 0 到阶段 6 已完成。后续可以继续添加功能，但应沿用以下边界：

```text
页面/组件层：app/pages、app/components，只负责 UI 编排和交互
前端状态层：app/composables，负责页面状态、表单状态、API 调用编排
API handler：server/api，只做鉴权、参数校验、调用 service、返回响应
业务服务层：server/services，承载领域规则和跨接口复用逻辑
数据访问边界：Prisma 查询、RAG raw SQL、文件系统访问集中在对应 service/repository
DTO/API 契约：types/api.ts、types/dto/**
```

当前关键架构约束：

- 后台 API 必须调用 `requireAdmin(event)`；已有测试会扫描 `server/api/admin/**/*.ts`。
- 普通登录用户权限走 `requireUser(event)`；管理员判断集中在 `server/services/security/access-control.service.ts`。
- 文章写入规则集中在 `server/services/posts/post.service.ts`。
- 文章公开查询口径集中在 `server/services/posts/post-query.service.ts`，公开文章条件是 `PUBLISHED + publishedAt <= now`。
- 分类、标签规则集中在 `server/services/taxonomy/taxonomy.service.ts`。
- 菜单规则集中在 `server/services/menus/menu.service.ts`。
- 设置默认值、公开/后台字段边界、输入校验集中在 `server/services/settings/settings.schema.ts` 和 `settings.service.ts`。
- 上传、图库、上传路径安全集中在 `server/services/media/**`。
- 认证、限流、角色策略集中在 `server/services/security/**`。
- 公开内容缓存响应头集中在 `server/utils/cache-control.ts`，当前采用 HTTP 短缓存策略，不使用 Nitro server cache。
- AI 功能按 `client/config/schema/parser/prompt/feature service/error` 分层，新增 AI 能力应补 schema 和 parser 测试。
- RAG raw SQL 只能出现在 `server/services/rag/post-chunk.repository.ts`，已有测试约束 `$queryRawUnsafe` / `$executeRawUnsafe` 边界。

## 重要维护规则

### 数据库表结构变更必须写 migration

涉及数据库表、字段、索引、枚举、关系、扩展的任何变更，必须同时维护：

```text
prisma/schema.prisma
prisma/migrations/<timestamp>_<name>/migration.sql
```

不要只改 `schema.prisma`。生产环境执行的是：

```bash
npx --package prisma@6 prisma migrate deploy
```

它只会执行 `prisma/migrations/**/migration.sql`，不会根据 `schema.prisma` 自动补表。

本项目曾出现过 `schema.prisma` 里已有 `Setting` model，但 migration 没创建 `"Setting"` 表，导致生产环境报错：

```text
The table `public.Setting` does not exist in the current database.
```

修复 migration：

```text
prisma/migrations/20260701150000_add_missing_setting_table/migration.sql
```

### 生产库禁止用 db push 代替 migration

生产环境不要用：

```bash
prisma db push
```

生产统一使用：

```bash
docker compose -f docker-compose.server.yml exec -T app npx --package prisma@6 prisma migrate deploy
```

### pgvector 迁移要求

`20260704143000_add_post_chunks_pgvector` 会执行：

```sql
CREATE EXTENSION IF NOT EXISTS vector;
CREATE INDEX "PostChunk_embedding_hnsw_idx" ON "PostChunk" USING hnsw ("embedding" vector_cosine_ops);
```

生产数据库必须支持 pgvector。当前 `docker-compose.server.yml` 已使用：

```text
pgvector/pgvector:pg16
```
